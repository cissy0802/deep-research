# 席位 A:反证搜索

## G1 AGENTbench 成功率 — verdict: **contradicted**

**独立证实**:
- [独立] Lulla et al., 'On the Impact of AGENTS.md Files on the Efficiency of AI Coding Agents', arXiv:2601.20404 (Jan 2026, ICSE 2026 JAWs workshop; authors incl. Mohsenimofidi, Galster, Zhang, Baltes, Treude) — https://arxiv.org/abs/2601.20404
  - Partially agrees on ONE component: with developer-committed AGENTS.md files (10 repos, 124 real PRs, Codex/Claude Code agents), task completion behavior was 'comparable' with vs without the file — consistent with ETH's core claim that context files do not generally improve success rates (though not with the +4% developer-written figure). Different team, different data/measurement.
- [独立] Shepard & Albrecht, 'Probe-and-Refine Tuning of Repository Guidance for Coding Agents', arXiv:2606.20512 (Jun 2026) — https://arxiv.org/abs/2606.20512
  - Weakly corroborates the ETH ablation nuance (LLM-generated guidance can be net-positive when it adds non-redundant information): their one-shot LLM-generated 'static KB' guidance gave +2.8pp over no-context on SWE-bench Verified, close to ETH's +2.7% no-README ablation flip. Independent team (Williams College) and setup (Qwen3.5-35B-A3B, 4 trials).

**矛盾证据**:
- [direct-measurement] Lulla et al., arXiv:2601.20404 'On the Impact of AGENTS.md Files on the Efficiency of AI Coding Agents' (Jan 2026) — https://arxiv.org/abs/2601.20404 ; also https://conf.researchr.org/details/icse-2026/jaws-2026-papers/31/
  - Directly contradicts the COST component of the finding ('raising inference cost >20%'): across 10 repos / 124 GitHub PRs with developer-committed AGENTS.md, presence of the file was associated with 28.64% LOWER median runtime and 16.58% LOWER output-token consumption (Wilcoxon p<0.05), with comparable task completion. A 2026 survey (iwoszapar.com/p/agents-md-do-context-files-help) explicitly flags these two papers as reaching 'opposite conclusions'. Concurrent independent work — 'Lulla' does not appear in the Gloaguen paper text.
- [direct-measurement] Shepard & Albrecht, 'Probe-and-Refine Tuning of Repository Guidance for Coding Agents', arXiv:2606.20512 (Jun 2026) — https://arxiv.org/abs/2606.20512
  - Contradicts the headline SUCCESS-RATE component ('LLM-generated context files reduce success rates'): on SWE-bench Verified (repos with normal READMEs/docs intact), an LLM-generated guidance file (tree-sitter repo map + one-shot generic guidance) raised mean resolve rate from 25.5% (unguided) to 28.3%, and iteratively refined guidance to 33.0% (4 independent trials; refined-vs-unguided p<0.001 in all trials, refined-vs-static p<0.05). Caveat: guidance content (structured repo map) differs from typical /init output, and agent/model differ.
- [indirect] BootstrapAgent, arXiv:2605.15815 (2026) — https://arxiv.org/abs/2605.15815
  - Indirectly contradicts the cost claim: reusable repo-knowledge '.bootstrap' contracts consumed by downstream Claude Code agents REDUCED token usage by 25.9% and build time by 22.3% — persisted repo context lowering, not raising, inference cost.
- [indirect] Learn-by-Interact (Google), arXiv:2501.10893 (Jan 2025) — https://arxiv.org/html/2501.10893v1
  - Older adjacent literature contradicting the general 'repo context added to agents hurts' framing: synthesized repository-interaction context provided via in-context learning improved agent performance by up to 19.5% across SWE-bench/WebArena/OSWorld. Construct is adjacent (synthesized experience data, not static AGENTS.md), so indirect.
- [opinion] Practitioner/expert critiques: Patrik Björklund, 'A Paper Says AGENTS.md Hurts Performance. My Setup Says Otherwise' — https://pbjorklund.com/articles/agents-md-paper-vs-practice/ ; HN threads https://news.ycombinator.com/item?id=47034087 and 47295454 (sample-selection pushback, answered in-thread by co-author Niels Mündler); dev.to 'Don't Ditch AGENTS.md — Fix What's In It' — https://dev.to/dortort/dont-ditch-agentsmd-fix-whats-in-it-24ph
  - Methodological critiques of external validity: paper tested only fresh, flat, monolithic context files (not aged files or hierarchical/dynamically-loaded setups); only Python OSS repos (closed-source codebases with un-inferable domain knowledge are where practitioners claim the payoff); measured only test-pass rate, not style/maintainability conformance. No counter-numbers provided — opinions.

**Notes**: Verdict rationale: two independent direct measurements contradict components of the finding. (1) Lulla et al. arXiv:2601.20404 (different team: Treude/Baltes/Zhang et al.; different data: 124 real PRs on 10 repos with developer-committed AGENTS.md) found LOWER runtime (−28.64%) and LOWER output tokens (−16.58%) with AGENTS.md present — the opposite sign of Gloaguen's '>20% inference cost increase'. Important caveats: neither is an exact replication — Lulla measured tokens/runtime under a different agent/task regime (real PR tasks vs benchmark issue resolution; cost accounting methods differ), and Shepard's guidance file includes a tree-sitter repo map unlike typical /init output; both results are arguably reconcilable with Gloaguen's own ablation (context files help when non-redundant with repo docs). So the headline claim ('context files tend to reduce success AND raise cost >20%') is contradicted in both components by independent measurements, but the nuanced version ('redundant, verbose LLM-generated files add cost without benefit; non-redundant guidance helps') is broadly consistent across all three papers. ## G2 Codex 效率 — verdict: **contradicted**

**独立证实**:
- [独立] Shepard & Albrecht, "Probe-and-Refine Tuning of Repository Guidance for Coding Agents" (arXiv 2606.20512, June 2026), https://arxiv.org/abs/2606.20512
  - Independent team/data (SWE-bench Verified, Qwen3.5-35B-A3B): iteratively tuned guidance files raise resolve rate 25.5%→33.0% vs unguided. Corroborates only the weaker claim that repository guidance files change agent behavior beneficially; it measures success rate, NOT runtime or output-token efficiency, so it does not corroborate the specific -28.64% runtime / -16.58% token finding.
- [非独立] Context Studios blog "AGENTS.md: The Research-Backed Guide to Making AI Agents 29% Faster", https://www.contextstudios.ai/blog/agentsmd-the-research-backed-guide-to-making-ai-agents-29-faster
  - Restates the Lulla et al. numbers (29% faster) with no new measurement — merely cites the original paper.
- [非独立] AkihikoWatanabe paper_notes issue #4775, https://github.com/AkihikoWatanabe/paper_notes/issues/4775
  - Reading note summarizing the original paper; no independent measurement.

**矛盾证据**:
- [direct-measurement] Gloaguen, Mündler, Müller, Raychev, Vechev (ETH Zurich SRI + LogicStar.ai), "Evaluating AGENTS.md: Are Repository-Level Context Files Helpful for Coding Agents?" (arXiv 2602.11988, Feb 13 2026), https://arxiv.org/abs/2602.11988 ; benchmark: https://github.com/eth-sri/agentbench
  - Directly contradicts the efficiency direction: on 438 tasks (SWE-bench Lite + their AGENTbench of 138 tasks from 12 niche Python repos with developer-committed context files), providing AGENTS.md/context files INCREASED inference cost by >20% on average and INCREASED reasoning tokens by 14–22% (GPT-5.2) / 2–14% (GPT-5.1 mini) — reasoning tokens are output tokens, versus Lulla's -16.58% output tokens. LLM-generated files also cost +23% and reduced success ~2–3%; human-written files gave only ~4% success gain. Different team, different benchmark, different agent scaffold, same construct (AGENTS.md effect on agent efficiency), opposite conclusion.
- [direct-measurement] Systima engineering blog, "Claude Code Sends 4.7x More Tokens Than OpenCode Before Reading Your Prompt", https://systima.ai/blog/claude-code-vs-opencode-token-overhead
  - Practitioner direct measurement of the cost side Lulla did not count: adding a 72KB production AGENTS.md added ~20,000 input tokens to EVERY request in both harnesses (OpenCode 13,152→33,336; Claude Code 39,005→59,243). Contradicts the blanket 'AGENTS.md makes agents cheaper' reading, though it measures input-token overhead, not output tokens or runtime.
- [indirect] heise online, "AGENTS.md: Helpful agent briefing or token hog?", https://www.heise.de/en/background/AGENTS-md-Helpful-agent-briefing-or-token-hog-11245317.html ; also iwoszapar.com survey "Context Engineering Papers: What 20+ Studies Actually Say", https://www.iwoszapar.com/p/agents-md-do-context-files-help
  - Third-party analyses that explicitly frame Lulla et al. and Gloaguen et al. as reaching opposite conclusions on AGENTS.md efficiency, and reconcile them: Gloaguen counted total inference tokens including the context file on verbose/redundant files; Lulla counted runtime and output tokens on focused developer-style files. Both note the finding does not generalize to typical (often auto-generated, redundant) AGENTS.md files.

## G3 遵从率因子实验 — verdict: **single-source-unchallenged**

**独立证实**:
- [独立] Cobus Greyling, 'Instruction Fade-Out Is the Silent Killer of AI Agents' (https://cobusgreyling.substack.com/p/instruction-fade-out-is-the-silent), 2026
  - Corroborates ONLY the within-session decay sub-claim, directionally: measured format-rule compliance over a 20-turn agent conversation — 100% compliance turns 1-5, dropping to 80% in turns 6-15 as accumulated history dilutes system-prompt influence. Different team, different data (chat-agent format rules, not Claude Code config files); small n; does not test any of the four structural variables.
- [独立] Laban et al., 'LLMs Get Lost in Multi-Turn Conversation', arXiv 2505.06120 (referenced via https://cobusgreyling.substack.com/p/ai-agents-and-the-lost-in-conversation)
  - 200,000+ simulated conversations, 15 frontier LLMs: average ~39% performance drop when a task is spread across turns vs a single fully specified prompt — consistent with the direction of within-session degradation, but measures task performance in chat, not config-file instruction compliance in a coding agent.
- [独立] 'Getting Better at Working With You: Compiling User Corrections into Runtime Enforcement for Coding Agents', arXiv 2606.13174 (https://arxiv.org/pdf/2606.13174)
  - Independent measurement of coding-agent rule compliance: plain rules in context ('All Rules') satisfied only 55.0% of applicable preferences vs 31.6% with no rules and 70.1% with compiled/runtime enforcement. Loosely consistent with McMillan's theme that WHAT the file says structurally matters less than the delivery/enforcement mechanism and task context, but it does not test file size, position, nesting, or conflicts — so it corroborates the broader picture, not the specific factorial nulls.
- [独立] ETH Zurich, 'Evaluating AGENTS.md: Are Repository-Level Context Files Helpful for Coding Agents?', arXiv 2602.11988 (coverage: https://www.infoq.com/news/2026/03/agents-context-file-value-review/, https://www.marktechpost.com/2026/02/25/new-eth-zurich-study-proves-your-ai-coding-agents-are-failing-because-your-agents-md-files-are-too-detailed/)
  - 438 SWE-bench-Lite/AGENTbench tasks: LLM-generated context files reduce task success ~3% vs no context file and inflate inference cost >20%; human-written ones give only ~4% marginal gain. Consistent in spirit with 'practitioner assumptions about context-file craft are overstated', but the dependent variable is task success/cost, not instruction compliance, and it manipulates presence/origin of the file, not McMillan's four structural variables.

**矛盾证据**:
- [opinion] Anthropic Claude Code official docs / best-practices guidance (https://code.claude.com/docs/en/memory; echoed at https://www.humanlayer.dev/blog/writing-a-good-claude-md and https://dev.to/nishilbhave/claudemd-best-practices-the-complete-2026-guide-435j)
  - Contradicts the file-size null: vendor guidance states CLAUDE.md files over ~200 lines 'consume more context and reduce adherence'. No published measurement backs the claim, so it is an assertion, not data — but it is the vendor of the system under test disagreeing with the affirmative Bayesian null for size (BF10 0.05-0.10).
- [indirect] IFScale — Jaroslawicz et al. (Distyl AI), 'How Many Instructions Can LLMs Follow at Once?', arXiv 2507.11538 (https://arxiv.org/abs/2507.11538; https://distylai.github.io/IFScale/)
  - In tension with the size and position nulls, in a different regime: 20 frontier models show compliance degrading with instruction count (best model 68% at 500 instructions; threshold/linear/exponential decay patterns) plus a primacy bias toward earlier instructions. Direct measurement, but of instruction DENSITY in a single-prompt writing task — not file size in lines around one target instruction in an agentic session. Notably, IFScale's primacy bias is near zero at low densities, which partially reconciles it with McMillan's position null for typical CLAUDE.md scale.
- [indirect] 'Control Illusion: The Failure of Instruction Hierarchies in LLMs', arXiv 2502.15851 (https://arxiv.org/html/2502.15851v4)
  - In tension with the conflict null: with simple contradictory instructions (length, capitalization), models exhibit highly inconsistent behavior in which instruction they follow — i.e., conflicts demonstrably perturb behavior in chat settings. Different setup (system-vs-user prompt hierarchy, not conflicting instructions in an adjacent AGENTS.md), so it does not directly refute the affirmative null for adjacent-file conflicts on a specific target annotation.
- [indirect] Liu et al., 'Lost in the Middle' (2023, TACL) and 2026 practitioner follow-ups (https://dev.to/thousand_miles_ai/the-lost-in-the-middle-problem-why-llms-ignore-the-middle-of-your-context-window-3al2)
  - In tension with the position null: well-replicated position bias (start/end of context attended more than middle) that reportedly persists on 1M-token models in 2026. However, it was measured on long-context retrieval, not a 25-500-line config file at the top of an agent session, so the regimes differ by orders of magnitude.
- [indirect] Vercel, 'AGENTS.md outperforms skills in our agent evals' (https://vercel.com/blog/agents-md-outperforms-skills-in-our-agent-evals)
  - Mildly in tension with the 'structure/wording of the file doesn't matter' reading: Vercel's evals found small wording changes in AGENTS.md produced large behavior swings (e.g., 'You MUST invoke the skill' phrasing flipped outcomes; pass rate 53%→79%→100% across configurations). This measures phrasing/content-architecture effects, not McMillan's four variables, and their headline (static AGENTS.md context is reliably consumed every turn) partly supports McMillan's decay framing.

**Notes**: The core headline finding — the affirmative nulls for file size (25-500 lines), instruction position, single-vs-nested architecture, and adjacent-file contradictions on compliance in real Claude Code sessions — has, as of 2026-07, no direct replication, no direct-measurement contradiction, and no substantive expert critique that I could find (no HN thread, no OpenReview/alphaXiv discussion; only auto-poster and AI-summary sites reference the paper, arXiv 2605.10039 appeared ~2 months ago). The 'task identity dominates' and 'structure matters less than practitioners assume' themes are loosely consonant with the ETH Zurich AGENTS.md study (different DV: task success/cost) and the Compiled Rules paper (delivery mechanism, not structure, moves compliance 55%→70%). Net: a genuinely single-source result whose specific factorial nulls remain untested by anyone else; adjacent literature pulls in both directions but never measures the same thing. ## G4 2303 文件普查 — verdict: **independently-corroborated**

**独立证实**:
- [独立] Santos, Costa, Montandon & Valente (UFMG Brazil), "Decoding the Configuration of AI Coding Agents: Insights from Claude Code Projects", arXiv:2511.09268, IWAE 2026 (https://arxiv.org/abs/2511.09268, https://dl.acm.org/doi/10.1145/3786167.3788412)

## G5 Context rot — verdict: **independently-corroborated**

**独立证实**:
- [独立] LongMemEval original paper — Wu et al., ICLR 2025, arXiv:2410.10813 (UCLA et al., published Oct 2024, months before Chroma's report)
  - Long-context LLMs lose 30-60% accuracy when reading full ~115k-token chat histories vs oracle evidence sessions (GPT-4o: 0.924 with oracle vs 0.640 full-context) — the exact focused-vs-full-history effect Chroma reports, measured first by the benchmark's own creators.
- [独立] "Context Length Alone Hurts LLM Performance Despite Perfect Retrieval" — Du et al., EMNLP 2025 Findings, arXiv:2510.05381 (UIUC/Amazon/Argonne/HUJI team)

## G1 AGENTbench 成功率 — survivesAsEvidence: **False** [high]

**Fatal flaws**:
- No inferential statistics whatsoever, on effects smaller than known run noise: the paper contains zero confidence intervals, significance tests, or variance estimates (full-text grep confirms), and samples exactly ONE completion per instance ('We sample completions for each agent once', §4.1). The load-bearing deltas are -0.5pp on 300 tasks (~1.5 tasks) and -2pp on 138 tasks (~2.8 tasks) — below the 1-3pp run-to-run variance documented for agentic SWE-bench evals even at temperature 0 (and Qwen3-30B was run at temp 0.7). The directional claim 'context files tend to reduce success' rests on drops in 5 of 8 cells — 3 of 8 cells moved the other way.
- Severely underpowered for the claimed effect sizes: with n=138 single-run binary outcomes, the minimum detectable difference in paired proportions is roughly 8-10pp; a 2pp or 4pp delta is unresolvable. Asserting direction ('reduce', 'improve') from these point estimates is false precision; the design can only support 'no large effect detected'.
- Clustering ignored — effective n for the developer-written-file claim is 12, not 138: all AGENTbench instances come from 12 repositories, so there are only 12 distinct developer-written context files; instances within a repo share the file, toolchain, difficulty profile, and LLM-test style, yet all rates treat 138 instances as independent. The '+4% developer-written' average also masks a sign flip (negative for Claude Code) across only 4 model configs. SWE-bench Lite is majority-django; the paper's own appendix (Fig. 12) concedes per-repository estimates are 'noisy'.
- Circular, LLM-in-the-loop benchmark construction by the evaluated agent: AGENTbench was built using Codex + GPT-5.2 — an LLM agent filtered PRs, set up environments, rewrote task descriptions from the PR + golden patch, and generated the unit tests from the golden patch — and Codex/GPT-5.2 is then one of the evaluated configurations. Selection and phrasing biases favor that config. Solution-leakage control was manual inspection of a 10% sample (~14 instances); test validity was checked only as fail-on-base/pass-on-golden-patch, so correct alternative implementations can fail the LLM-written tests — and this false-negative rate plausibly interacts with the treatment, since context files change solution style (the paper shows agents obey them).
- Dependent-variable/construct gap: 'helpful' is operationalized solely as single-shot pass rate on (partly LLM-generated) regression tests plus dollar cost. Context files' advertised benefits — style-guide adherence, build/PR-convention compliance, code quality, maintainability — are unmeasured, and the paper's own trace analysis shows the files work as instructed (more testing, linting, repo-specific tooling), with those extra steps booked purely as 'cost'. The headline '>20% cost increase' partly measures instructions being followed, not overhead waste.

**Non-fatal caveats**:
- Preprint (Feb 13, 2026), not peer-reviewed; single version (v1).
- Conflict of interest, disclosed only as affiliation: LogicStar.ai (co-authors Müller, Raychev; Vechev is a founder) is a commercial agent-evaluation/debugging startup — a contrarian, press-friendly finding and a new benchmark are marketing-adjacent assets. No agent-vendor COI, and no formal COI statement.
- Contamination unaddressed: SWE-bench Lite issues predate all evaluated models' training cutoffs (well-documented leakage/quality problems), and several 'niche' AGENTbench repos (transformers, ansible, wagtail, tinygrad, smolagents) are heavily represented in training data; no per-instance date-vs-cutoff analysis. The authors themselves note Python parametric knowledge 'might... nullify the effect of context files'.
- The quoted +2.7% docs-removed ablation excludes Claude Code entirely ('due to its high cost') — the flip is measured on 3 of 4 configs, and the widely-quoted 'developer +4%' figure comes from the intact-docs condition, not this ablation.
- Multiple-comparison exposure with no correction: ~20+ cells across 2 benchmarks x 4 configs x 2-3 settings plus generator-model, prompt, and docs-removal ablations; trace analyses are exploratory and post hoc.
- Scope: Python-only, 3 agent scaffolds / 4 model configs, one context-file per repo root; findings quoted as if about 'coding agents' generally. Cost figures are averages of ratios over heterogeneous configs (GPT-5.2 +34%, Sonnet-4.5 +16%). The NONE condition on AGENTbench deletes a developer file the rest of the repo's docs/workflow may assume.
- This 'AGENTbench' is unrelated to the 2023 Tsinghua AgentBench — a live mis-citation hazard.
- Primary source access: arxiv.org is policy-blocked at the proxy (CONNECT 403); full text was obtained from a verbatim PDF mirror (readwise-assets.s3.amazonaws.com .../1152.pdf, header 'arXiv:2602.11988v1 [cs.SE] 12 Feb 2026') and cross-checked against prior multi-mirror verification in /home/user/deep-research/research/agent-readme/03-verification-round2.md.

**Notes**: Ruling is on the load-bearing success-rate deltas as quoted (-0.5pp SWE-bench Lite / -2pp AGENTbench / +4% developer / +2.7% ablation flip): these do NOT survive as evidence of directional effects. They are single-run point estimates on 138-300 binary outcomes clustered in 11-12 repos, with no variance estimation, at magnitudes below ordinary agentic-eval run noise, on a benchmark partly authored by one of the evaluated systems. Quoting them to 0.1pp precision is false precision; even the paper hedges ('tend to', 'marginal'), and its abstract's directional framing outruns its own design. What DOES survive, real-but-weak: (1) the null — no consistent success-rate improvement from context files in this Python/single-shot setting — because a large positive effect would have been visible despite the noise; (2) the cost/steps increase (~+20%, +2.4-3.9 steps), which is sign-consistent across all 8+ cells and mechanistically corroborated (reasoning-token counts, tool-call traces); (3) the behavioral finding that agents follow context-file instructions. If the finding must be quoted, the defensible form is: 'one underpowered, non-peer-reviewed single-run study found no consistent success-rate benefit and ~20% higher inference cost; its specific pp-level deltas are within noise and its author-built benchmark has circularity and clustering problems.' Verified paper facts for the record: Gloaguen, Mündler, Müller, Raychev, Vechev (ETH Zurich CS + LogicStar.ai); agents = Claude Code/Sonnet-4.5, Codex/GPT-5.2 and GPT-5.1 Mini, Qwen Code/Qwen3-30B-Coder; AGENTbench = 138 instances distilled from 5,694 PRs across 12 repos, tests average 75% coverage of modified code (min 2.5%). ## G2 Codex 效率 — survivesAsEvidence: **True** [medium]


**Non-fatal caveats**:
- Population is conditioned on curated developer-written AGENTS.md files: pipeline filtered 132 prior-corpus repos → 89 single-root-AGENTS.md → 26 passing a content-quality taxonomy filter → random 10. The −28.64%/−16.58% estimate applies only to repos that already have a good AGENTS.md; it is NOT evidence that adding an AGENTS.md to an arbitrary repo yields these gains, and the quality filter plausibly makes it an upper bound. Fatal if quoted as a general 'AGENTS.md makes agents ~29% faster' claim.
- DV is OUTPUT tokens only, explicitly excluding input tokens — but AGENTS.md adds input tokens to every model request. The −16.58% figure is not evidence of net token/cost savings; the ETH SRI counter-study (arXiv 2602.11988, same agent family: Codex + GPT-5.2) found context files increase total inference cost by >20% on average with no general success-rate gain. Fatal if quoted as 'saves tokens/cost'.
- Clustering ignored: 124 PRs nested in 10 repositories, but the Wilcoxon signed-rank test treats pairs as independent. Effective sample size is closer to 10 than 124; p<0.05 is inflated, and per-repo heterogeneity (whether a few repos drive the median shift) is not reconstructable from available mirrors.
- Repetition and ordering protocol unverified: no evidence of multiple runs per PR-condition pair; agent runtimes/token counts are highly nondeterministic run-to-run, and wall-clock time of a proprietary cloud agent is confounded by API latency/load unless conditions were interleaved — unreported in accessible material. The runtime number (−28.64%) is the weaker of the two DVs; the token number partially corroborates it since generation time tracks output tokens.
- 'Comparable task completion behavior' — the guard that makes 'faster' meaningful rather than 'does less work' — has an operationalization I could not verify from any mirror (possibly unit tests, possibly weaker). If completion was assessed loosely, efficiency gains could partially reflect shallower patches.
- Training-data contamination: tasks are historical merged PRs from public GitHub repos, near-certainly in gpt-5.2-codex training data. Affects both arms equally (so the paired difference is less biased), but the absolute completion rates and the exploration-reduction mechanism may not transfer to unseen codebases.
- Single agent, single model snapshot (OpenAI Codex, gpt-5.2-codex — an agent that natively reads AGENTS.md), tiny tasks (median ~98.57s, ~2,925 output tokens without the file); no evidence for other agents, larger tasks, or later model versions. The G2 finding does correctly scope to Codex.
- Author-built benchmark derived from the authors' own prior corpus work; no independent replication yet; replication-package availability unverified. Venue is a workshop (JAWs@ICSE 2026) with lightweight peer review, and the paper has v1→v2 revisions (v2 dated ~2026-03-30).
- No stated multiple-comparison correction across the two headline DVs (minor given both are primary and directionally consistent).
- Primary source inaccessible through this proxy (arXiv, author PDF, Semantic Scholar all 403); methodology reconstructed from 3+ independent mirrors and search snippets quoting the paper verbatim, plus this workspace's earlier verification cache — specific unverified items are flagged above.

**Notes**: Ruling: real-but-weak; survives only as narrowly scoped. ## G5 Context rot — survivesAsEvidence: **True** [medium]
